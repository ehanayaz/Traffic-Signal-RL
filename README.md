# Traffic-Signal-RL

A reinforcement learning project that trains an AI agent to control a single traffic intersection using a Deep Q-Network (DQN) and the [SUMO](https://sumo.dlr.de/) traffic simulator.

## Project Overview

The agent observes the number of waiting vehicles on each of the four approach lanes of a single 4-way intersection, as well as the current traffic-light phase. Every 10 simulation seconds it decides whether to **keep** the current green phase or **switch** to the next one. The reward at each step is the negative total queue length — the fewer cars waiting, the higher the reward.

After training (`main.py`) the learned weights are saved to `traffic_brain.pth`. They can then be loaded by `test.py` to watch the trained agent control the intersection live in the SUMO graphical interface.

## File Descriptions

| File | Purpose |
|------|---------|
| `model.py` | Defines the **Deep Q-Network** (DQN) — a 3-layer fully-connected neural network built with PyTorch. Takes a 5-element state vector as input and outputs Q-values for the 2 possible actions (keep / switch). |
| `agent.py` | Implements the **DQN Agent** with an experience-replay buffer. Handles epsilon-greedy exploration, stores transitions in memory, and trains the network with random mini-batches via the `replay()` method. |
| `main.py` | **Training script.** Runs 50 episodes of headless SUMO simulation. At each decision step it calls `get_state()`, asks the agent for an action, executes it in the simulation, measures the reward with `get_reward()`, and stores the transition for learning. Saves the final model to `traffic_brain.pth`. |
| `test.py` | **Evaluation / visualization script.** Loads the trained weights from `traffic_brain.pth` and runs the agent in `sumo-gui` (the graphical SUMO interface) for 3 600 simulation steps with a slight real-time delay so you can watch the agent in action. |
| `single.net.xml` | SUMO **network file** — describes the road network: a single 4-way intersection (`A0`) with four 200 m approach roads (North, South, East, West) and a 4-phase traffic-light controller. Generated by SUMO `netgenerate`. |
| `single.rou.xml` | SUMO **route file** — pre-computed vehicle routes generated by `duarouter` from `trips.trips.xml`. Vehicles depart every ~2 seconds for the full 3 600-second simulation. |
| `single.sumocfg` | SUMO **configuration file** — ties together the network and route files and sets the simulation time window (0 – 3 600 s). Referenced by both `main.py` and `test.py`. |
| `trips.trips.xml` | Raw **trip definitions** generated by SUMO's `randomTrips.py` (period = 2.0 s). Used as input to `duarouter` to produce `single.rou.xml`. |
| `traffic_brain.pth` | Pre-trained **PyTorch model weights** — ready to load directly into `test.py` without re-running training. |

## Architecture

```
State (5 inputs)
  ├── Queue length — Lane 1 (North)
  ├── Queue length — Lane 2 (East)
  ├── Queue length — Lane 3 (South)
  ├── Queue length — Lane 4 (West)
  └── Current traffic-light phase (0–3)
           │
           ▼
    ┌─────────────┐
    │  FC  5 → 64 │  ReLU
    │  FC 64 → 64 │  ReLU
    │  FC 64 →  2 │  (linear)
    └─────────────┘
           │
           ▼
Actions (2 outputs)
  ├── 0 — Keep current green phase
  └── 1 — Switch to next phase (yellow → new green)
```

**Key hyperparameters** (defined in `agent.py` / `main.py`):

| Parameter | Value | Description |
|-----------|-------|-------------|
| `EPISODES` | 50 | Number of training episodes |
| `SIMULATION_STEPS` | 1 000 | Steps per episode |
| `ACTION_DELAY` | 10 | Simulation steps between decisions |
| `MAX_MEMORY` | 10 000 | Replay buffer size |
| `BATCH_SIZE` | 64 | Mini-batch size for training |
| `LR` | 0.001 | Adam learning rate |
| `GAMMA` | 0.95 | Discount factor |
| `epsilon` | 1.0 → 0.01 | Exploration rate (decays × 0.995 per step) |

## Prerequisites & Installation

### 1 — Install SUMO

SUMO (Simulation of Urban MObility) must be installed **and** the `SUMO_HOME` environment variable must be set.

**Ubuntu / Debian**
```bash
sudo add-apt-repository ppa:sumo/stable
sudo apt update
sudo apt install sumo sumo-tools sumo-doc
echo 'export SUMO_HOME="/usr/share/sumo"' >> ~/.bashrc
source ~/.bashrc
```

**macOS (Homebrew)**
```bash
brew tap dlr-ts/sumo
brew install sumo
echo 'export SUMO_HOME="$(brew --prefix)/share/sumo"' >> ~/.zshrc
source ~/.zshrc
```

**Windows**

Download the installer from [sumo.dlr.de/docs/Installing/index.html](https://sumo.dlr.de/docs/Installing/index.html) and add the `bin` directory to `PATH`, then set `SUMO_HOME` to point to the installation folder.

### 2 — Install Python dependencies

Python 3.8 or newer is required.

```bash
pip install torch numpy
```

The `traci` Python package ships as part of SUMO — make sure `$SUMO_HOME/tools` is on your `PYTHONPATH`:

```bash
export PYTHONPATH="$SUMO_HOME/tools:$PYTHONPATH"
```

You can add this line to your shell profile (`~/.bashrc`, `~/.zshrc`, etc.) to make it permanent.

### 3 — Clone the repository

```bash
git clone https://github.com/ehanayaz/Traffic-Signal-RL.git
cd Traffic-Signal-RL
```

## Running the Project

All commands should be run from the repository root directory.

### 1. Train the Agent

```bash
python main.py
```

This runs 50 headless (no GUI) SUMO episodes. Progress is printed after each episode:

```
Episode: 1/50 | Total Reward (Score): -4823 | Epsilon: 0.97
Episode: 2/50 | Total Reward (Score): -4201 | Epsilon: 0.94
...
Training complete. Model saved as traffic_brain.pth!
```

The reward (score) should trend upward as the agent learns. Training typically takes a few minutes depending on your hardware.

### 2. Test / Visualize the Agent

After training (or using the included pre-trained `traffic_brain.pth`):

```bash
python test.py
```

This opens the SUMO graphical interface and lets the trained agent control the intersection for 3 600 simulation seconds. The agent runs purely in exploitation mode — no random actions.

> **Tip:** In the SUMO-GUI toolbar, click the green **Play** button (or press `D`) to start the simulation.

## Configuration & Tuning

- **Change the number of training episodes:** Edit `EPISODES` at the top of `main.py`.
- **Change episode length:** Edit `SIMULATION_STEPS` in `main.py`.
- **Adjust network size:** Change the `hidden_size` argument in `DQN(...)` calls in `agent.py` and `test.py`.
- **Regenerate traffic:** Use SUMO's `randomTrips.py` to create new trips, then run `duarouter` to produce a fresh `single.rou.xml`.

## How It Works

1. **Observation** — At each decision step, `get_state()` queries SUMO via TraCI for the number of halting vehicles on each controlled lane and the current traffic-light phase. This produces a 5-element numpy array.

2. **Action selection** — During training the agent uses an ε-greedy policy: with probability ε it picks a random action (explore); otherwise it feeds the state through the DQN and picks the action with the highest Q-value (exploit). ε decays from 1.0 to 0.01 over time.

3. **Action execution** — Action `0` does nothing (keeps the current green). Action `1` triggers a 3-second yellow phase and then switches to the opposing green phase.

4. **Reward** — `get_reward()` returns the negative sum of all halted vehicles. The agent is incentivized to minimize queues across all lanes simultaneously.

5. **Learning** — Each transition `(state, action, reward, next_state, done)` is stored in a fixed-size replay buffer. Once 64 transitions are collected, the `replay()` method samples a random mini-batch and updates the network using the Bellman equation and MSE loss with the Adam optimiser.
